# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## Apply changes

```bash
# Short alias (defined in home/shell.nix)
rebuild

# Full command
cd ~/nixos-dotfiles && git add . && sudo nixos-rebuild switch --flake .#nixos
```

Other useful commands:

```bash
nix flake check          # validate the flake without building
nix flake update         # bump all inputs (nixpkgs, home-manager)
nixos-rebuild dry-run --flake .#nixos   # preview what would change
```

## Architecture

This is a single-host NixOS flake. `flake.nix` is the only entry point — it wires together the system config and home-manager as a NixOS module (not as a standalone flake output).

| Path | Purpose |
|---|---|
| `hosts/nixos/default.nix` | System-level config: boot, hardware, desktop (KDE/Plasma 6), NVIDIA, audio, gaming, system packages |
| `hosts/nixos/hardware.nix` | Auto-generated hardware config — avoid editing manually |
| `home/default.nix` | Home-manager entry; imports the other `home/` modules |
| `home/gtk.nix` | GTK dark theme (adw-gtk3-dark) and dconf settings |
| `home/shell.nix` | Bash config and shell aliases |
| `home/vim.nix` | Vim setup managed by home-manager (`programs.vim`) |
| `assets/` | Static files referenced by Nix (e.g. Limine bootloader wallpaper) |

## Key conventions

- **System packages** go in `hosts/nixos/default.nix` under `environment.systemPackages`.
- **User programs with config** go in a dedicated `home/<name>.nix` module, then imported in `home/default.nix`.
- `home-manager.useGlobalPkgs = true` — home-manager shares the system nixpkgs, so do not add a separate `pkgs` input.
- The flake targets `nixos-25.11`; keep home-manager pinned to `release-25.11` to match.
- `hardware.nix` is generated by `nixos-generate-config` — regenerate it there, then copy here if hardware changes.
